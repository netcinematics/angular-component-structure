Rationale

This boilerplate provides multiple ways to use Components in an Architecture. Some of the benefits are:
  1. Insight into reusability, portability, modularity, and amount of code.
  2. Component Structure, shows advantages of group by feature
  3. Also shows advantages of group by type with a common object.
  4. 'Adjacency' advantages for tests and templates.
  5. 'Composition' advantages for building a heirarchy of objects.
  6. Is a great boilerplate for starting exmperiments like testing solutions.
  7. Also a good place to prove responsive views.
